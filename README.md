# Центр имени Мирослава — лендинг

Прототип сайта на чистом HTML / CSS / JS. Без сборки и зависимостей —
открывается двойным кликом по `index.html`.

## Структура

```
miroslav-center/
├── index.html              # вся разметка (одностраничный лендинг с якорями)
└── assets/
    ├── css/
    │   ├── reset.css        # нормализация
    │   ├── variables.css    # дизайн-токены: палитра, шрифты, отступы
    │   └── styles.css       # все стили по секциям
    ├── js/main.js           # бургер-меню, scroll-reveal, видео, карусель отзывов, форма
    ├── img/                 # logo.jpg, logo-mark.jpg, icon-whatsapp.svg, icon-max.svg, reviews/
    ├── videos/              # видеоролики для секции «Наши специалисты»
    └── fonts/               # сюда локальные шрифты, если уходить от Google Fonts
```

## Секции

Шапка → Hero → О центре (счётчики + цитата) → Наши специалисты (8 видео) →
Мастера (4, сетка 2×2) → Отзывы (карусель) → Услуги (01–06) →
С какими проблемами мы работаем (8 категорий) → Цитата основателя →
Запись + контакты → Футер.

### С какими проблемами мы работаем

Сетка из 8 категорий-карточек со списками направлений (на десктопе 2 колонки,
на мобильном 1). Маркеры списков — золотой ромб (`.problem-card__list li::before`).
Тексты правятся прямо в `index.html` (секция `id="problems"`). Каждая карточка
`.problem-card` — самодостаточный повторяемый блок для WordPress (ACF-репитер
с вложенным списком).

### Мессенджеры (WhatsApp / MAX)

Иконки стоят в шапке и в футере (`assets/img/icon-whatsapp.svg`,
`assets/img/icon-max.svg`).

- **WhatsApp** — подключён: `https://wa.me/79852867618` (открывает чат).
- **MAX** — ссылка пока не подставлена. ⚠️ У MAX **нет** ссылок по номеру
  телефона (как `wa.me`) — ради приватности. Рабочая ссылка на личный аккаунт
  только вида `https://max.ru/u/[ХЕШ]`, и её надо взять **в приложении MAX**:
  профиль (аватар слева вверху) → иконка QR-кода → «Поделиться» → скопировать
  ссылку. Полученный `https://max.ru/u/...` вставить в `href` обоих блоков
  `.messenger-icon` для MAX (в шапке и футере) — место помечено комментарием.

### Отзывы (карусель)

10 карточек; на десктопе видно по 4, на планшете 2, на мобильном 1.
Прокрутка стрелками (≥600px) и точками-страницами; работает свайпом.
Рейтинг — 4–5 звёзд (приглушённая звезда = `.review-card__star--off`).
Аватары — `assets/img/reviews/r1…r10.jpg` (плейсхолдеры реальных лиц;
заменяются на свои). Тексты и имена правятся прямо в `index.html`.

### Наши специалисты (видео)

Галерея вертикальных роликов (9:16). Видео грузятся «лениво»
(`preload="metadata"` — тянутся только заголовки, а не весь файл),
полный ролик скачивается только по клику на золотую кнопку play.
При запуске одного ролика остальные ставятся на паузу.

- **Файлы** — `assets/videos/*.MP4` (подключены в `index.html`).
- **Подписи специалистов** — сейчас карточки без имён. Чтобы добавить
  имя/специализацию под роликом, вставьте после блока `.specialist-card__media`
  свой `<div class="specialist-card__body">…</div>` (стиль можно взять
  с `.master-card__name` / `.master-card__role`).
- **WordPress** — это нативные теги `<video>`, ровно как у блока «Видео».
  Карточка `.specialist-card` — самодостаточный повторяемый блок: удобно
  ложится на ACF-репитер или Gutenberg-паттерн.

### Мистические элементы (атмосфера)

Все на чистом CSS/SVG, без зависимостей, отключаются при `prefers-reduced-motion`:

- **Hero** — вращающийся арканный круг-сигил за заголовком (`.hero__sigil`,
  inline-SVG) + парящие золотые частицы (`.hero__particles`).
- **Лунный разделитель** (`.moon-divider`) между «Отзывами» и «Услугами» —
  фазы луны от новолуния к убыванию на тонкой линии.
- **Мерцающие искры** (`.sparkles`) вокруг свечей в секции цитаты.
- **Специалисты** — звёздное поле (`.starfield`) + пульсирующее кольцо у кнопок
  play (`.specialist-card__play-icon::before`).
- **Услуги** — вращающийся лучистый орнамент-солнце (`.services__ornament`) на
  фоне + лёгкие звёзды.
- **Свечение карточек** при наведении (мастера / услуги / направления).

> Каждая секция получила свой мотив (сигил → свечи → луна → звёзды → солнце),
> чтобы тема была единой, но не повторялась. Интенсивность задаётся `opacity`
> у `.hero__sigil` / `.services__ornament` и количеством `<span>` в эффектах.

### 3D-свечи (CSS)

В секции с цитатой основателя — две свечи на чистом CSS: одна горит
(мерцающее пламя + тёплый ореол), вторая просто стоит (обугленный фитиль,
лёгкий дымок). Объём воска даёт горизонтальный градиент. Без WebGL и
зависимостей — переносится в WordPress как обычный HTML-блок. Анимации
отключаются при `prefers-reduced-motion`. Разметка — `.candles` в `index.html`,
стили — блок «3D-СВЕЧИ» в `styles.css`.

## Что заменить перед запуском

- **Телефон** `+7 (000) 000-00-00` — в секции «Контакты» (`index.html`).
- **Логотип** — большой `logo.jpg` в баннере-шапке (`.brand-banner`), компактная
  эмблема `logo-mark.jpg` в навбаре, полный `logo.jpg` в футере.
  Белый фон логотипа убран через **`mix-blend-mode: darken`** (сливается с
  кремовым фоном; `isolation: isolate` на баннере и навбаре делает приём
  стабильным). Это работает, потому что фон страницы светлый. **Лучшее
  решение на будущее** — попросить у клиента логотип в прозрачном **PNG/SVG**:
  тогда blend-mode не нужен и логотип будет корректен на любом фоне.
- **Название** — в логотипе «**Академия**», в тексте сайта «**Центр**».
  Определитесь с единым названием.
- **Фото мастеров** — сейчас инициалы в золотом круге; фото класть в `assets/img/`.
- **Форма** — фронт-валидация + экран «успеха». Бэкенда нет: в `main.js`
  (блок «Форма заявки») помечено место для `fetch()` на endpoint или
  отправки уведомления в Telegram.

## Дизайн

Палитра и шрифты выведены из шаблона клиента: кремовый фон, золото-акцент,
заголовки `Playfair Display`, текст `PT Serif` (оба с кириллицей).
Mobile-first, поддержка клавиатурной навигации и `prefers-reduced-motion`.
